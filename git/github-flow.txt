  - Github Flowとは

  - 説明
  Githubで開発をする際に使用するシンプルなワークフローのこと
  メインとなるmainブランチ、作業用のブランチを作成して迅速に開発を行うことができる。

  git-flowとの違いは？

  - 説明
  mainブランチ、作業用ブランチの2つしか使わないのが、Github Flowであるが、
  git-flowでは5つのブランチを駆使する。5つの中でも、「メインブランチ」と「サポートブランチ」の2種類に別れる。
  
  - 「メインブランチ」
  mainブランチ:リソース済みのソースコードを管理するブランチ
  
  developブランチ:次のリソースに備えて、最新のブランチを管理するブランチ

  - 「サポートブランチ」
  featureブランチ:新しい機能の開発や簡単なバグ修正などの開発作業を行うためのブランチ
  
  releaseブランチ:リリース作業を行うためのブランチ

  hot-fixブランチ:緊急の修正作業を行うためのブランチ

  2種類のみのブランチで運用するGitHub Flowに比べ、それぞれ役割をもつ5種類のブランチを使い分けるgit-flowは複雑で大規模な開発に適しているとされている。
